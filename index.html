What is ECMAScript?

ECMAScript (or ES) is a general-purpose programming language, standardised by Ecma International according to the document ECMA-262.ECMA stands for ' European Computer Manufacturer's Association', which is the maintaining association of JavaScript. It is a JavaScript standard meant to ensure the interoperability of web pages across different web browsers. ECMAScript is commonly used for client-side scripting on the World Wide Web, and it is increasingly being used for writing server applications and services using Node.js.

What is the relation between Ecmascript and JavaScript?

The different versions of JavaScript is given the name of ECMAScript.The implementation of the scripting language specified by ECMAScript is called JavaScript. JavaScript is a ever evolving programming language,which is evolved by ECMA international and named as ECMAScript.The versions of Ecmascript i.e JavaScript releases every year with new features.

Future of JavaScript

Though Javascript is an old Technology, just like our computer is. It was created a relatively long time ago and suffered some backlash a while back. But with the arrival of ECMAScript and Node.js things changed a lot, and JavaScript is nowadays the third most used language in the world with java and python above it. With the development of CSS and with the growing importance of Vue.js,Node.js and other frameworks and API driven web applications, it’s absolutely stupid to think that JavaScript has no future.

JavaScript is the most prime and trendiest languages currently. And since the body that controls the ECMAscript standard has moved to an annual release cycle, JavaScript has lots of wonderful updates and modern language features. It’s matured into a beautiful language day by day. The language will always have a historical cruft because web standards strive for 100% backward compatibility with every piece of code ever written for the web since it began.HTML, CSS and JavaScript don’t have the luxury of tossing out the old for this reason. With so many new best practices,you’ll never see most of that.
